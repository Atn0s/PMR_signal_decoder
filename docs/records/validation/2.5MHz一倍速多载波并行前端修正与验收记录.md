# 2.5 MHz 一倍速多载波并行前端修正与验收记录

日期：2026-07-15
适用分支：当前并行链路开发分支

## 1. 修正目标

本轮不改变 DMR、P25、dPMR、NXDN、TETRA 的协议判据，也不根据某个测试文件的
绝对时间调整 Probe 窗口。目标是修复以下结构性问题：

1. 固定 `-60 dB` 初始噪声底会把高于该值的纯噪声误判为文件起点活动；
2. 错误 Epoch 会提前消耗 DMR 的 1.5 秒 Probe 预算；
3. 锁定制式后的滑窗解码仍同步阻塞采集/UI；
4. 原入口一次只能维护一个选频链，无法同步处理三个已知载频；
5. 2.5 MS/s 与默认 120 kS/s 不构成整数抽取；
6. TETRA 训练搜索存在按比特位置逐点执行的 MATLAB 循环；
7. 0.25x 拉取式演示不能证明真实 1x 连续输入能力。

DMR 最大 Probe 窗口仍为 **1.5 秒**，TETRA 最大 Probe 窗口仍为 **6 秒**。

## 2. 活动检测修正

### 2.1 自适应噪声底

`radio.stream.defaultConfig().activity.initialNoiseFloorDb` 默认改为 `NaN`，表示由首个
可用 IQ 微批自动建立噪声底。每个微批最多取四个 2048 点 FFT 段；对复高斯噪声，
周期图中位数除以 `log(2)` 可估计平均噪声功率。窄带 PMR 信号只占所选基带的一部分，
因此即使信号从首样点开始，也不要求先等待一段静默校准。

若上游候选跟踪器或已标定接收机明确提供有限的 `initialNoiseFloorDb`，检测器继续把它
当作外部先验，兼容原有 PFB 路径和确定性测试。

活动结果新增：

- `snrDb`；
- `instantaneousNoiseFloorDb`；
- `smoothedNoiseFloorDb`；
- `spectralNoiseValid`；
- `noiseFloorSource`。

### 2.2 Epoch 与 Probe 时间基准

Probe 快照继续严格从 `epoch.candidateStartSample` 开始。前导静默只由活动检测器消费，
不计入某一制式的最大 Probe 窗口。回归覆盖：

- 连续 6 秒、约 `-47 dB` 的噪声：0 个 Epoch；
- 同一 4FSK 信号从 0、1.5、6.0 秒出现：候选起点分别为 0、1.5、6.0 秒；
- 原始三载波文件的实际结果：
  - `-300 kHz` DMR：1.500 秒建 Epoch，1.800 秒确认；
  - `+150 kHz` DMR：6.900 秒建 Epoch，7.200 秒确认；
  - `+600 kHz` P25：0.900 秒建 Epoch，1.000 秒进入确认窗口。

不存在“文件必须一开始就有信号”的假设，也没有写入 1.59 秒或 6.9 秒等样例专用参数。

## 3. 多载波与异步连续解码

新增：

```text
+radio/+tuned/multiStreamScannerInit.m
+radio/+tuned/multiStreamScannerFeed.m
+radio/+tuned/multiStreamScannerFinalize.m
```

一个宽带 `IqChunk` 同时扇出到 N 个已知大致载频：

```text
wideband IqChunk
  ├─ DDC(channel 1) -> Activity/Epoch -> five-protocol race -> winner decoder
  ├─ DDC(channel 2) -> Activity/Epoch -> five-protocol race -> winner decoder
  └─ DDC(channel N) -> Activity/Epoch -> five-protocol race -> winner decoder
```

每个通道独立保存：

- DDC/NCO/滤波状态；
- ActivityDetector 与 Epoch；
- Probe 状态、赢家与 generation；
- 连续解码状态、PDU 和积压样点；
- `channel_id`、目标频率和相对频偏元数据。

新增 `lockedDecoderStart/Poll/Cancel`，使锁定后的完整解码也使用进程池。采集调用栈只
提交或轮询任务，不等待 P25/TETRA 完成。若 worker 完成时实时边界已经前移，系统不再
反复执行“必须追到当前最新样点才 LOCKED”的追赶循环；而是立即锁定，并按样点顺序
处理有界积压。`backlogSamples` 与 `overrunSamples` 用来区分正常积压和环形缓冲覆盖。

## 4. DDC 采样率选择

新增 `radio.tuned.resolveInputConfig`：

- 61.44 MS/s 输入保持 120 kS/s，整数抽取 512；
- 2.5 MS/s 输入自动选择 125 kS/s，整数抽取 20；
- 输出奈奎斯特频率必须高于当前 55 kHz DDC 阻带边界；
- 找不到合法整数抽取时明确报错，不使用有理重采样偷偷替代实时 DDC。

## 5. TETRA 性能修正

协议窗口和确认判据未变化，优化仅消除重复计算：

1. 训练序列滑动汉明距离由逐位置 `nnz(xor(...))` 改为等价卷积公式；
2. π/4-DQPSK 四个等间距相位中心的最近误差用模 `pi/2` 折叠计算；
3. 长窗口相位偏移搜索用均匀分布的最多 8192 个 transition 选择网格点，选定后仍在
   全部 transition 上计算最终误差；
4. `swap_bits` 复用 `standard` 相位偏移，`conjugate_swap` 复用 `conjugate` 相位偏移。

原 6 秒 DMR 错配输入的 TETRA 路径约 4.42 秒，本轮实测为 **1.335 秒**，下降约 70%。
真实 TETRA 回归仍输出 685 条事件，训练序列卷积结果另与朴素 XOR 逐点实现逐项比较。

## 6. 去重语义

`radio_live_frontend` 默认 `Deduplicate=false`，满足调试阶段“每取得一块 PDU 就立即
显示”的要求。这里区分两类去重：

- **必须保留的窗口重叠抑制**：相同样点、相同内容因滑窗重解码再次出现时不重复输出；
- **暂时关闭的语义去重**：不同样点但内容相同的空口 PDU 不再合并。

CALL/SESSION 是派生状态而不一定是一块原始空口 PDU。关闭语义去重后，它们可能随解码
窗口重复生成，因此不同异步调度下总记录数可能略有变化。后续优化应把“原始 PDU”、
“状态变化事件”和“会话摘要”分成三种输出类型，而不是重新开启全局内容去重。

## 10. 当前边界

1. 已证明的是 **2.5 MHz、三个已知大致载频**；未证明 60 MHz MATLAB CPU 实时。
2. 频谱/file/DDC 调度仍在 MATLAB UI timer 所在线程；协议 Probe、catch-up 和 locked
   decode 已在进程 worker。真实 SDR 到位后仍需原生生产者、有界 CI16 队列和硬件时间戳。
3. 五制式任务共享固定进程池。载波数量继续增加时必须加入全局调度优先级，不能无限
   增加 worker。
4. 当前完整解码器仍是窗口式接口；真正最低 PDU 延迟需要逐制式实现
   `streamInit/feed/finalize`。
5. `overrunSamples>0` 必须视为实时失败；不能静默丢弃或用慢放掩盖。

## 11. Run/Clear 状态机修正（2026-07-15）

### 11.1 根因

旧 `Run decode` 回调按以下顺序执行：

```text
stop timer -> close file source -> warm DDC/process pool
           -> reopen file source -> reset PSD -> reset replay clock
```

因此用户看到的“卡顿后文件似乎重播”是确定性行为，不是显示假象。
`Clear carriers` 又把“`scanner` 字段非空”等同于“解码器正在运行”，已终止或
已 finalize 的对象也会被拒绝，从而抛出 `Stop the decoder before clearing carriers`。

### 11.2 新生命周期

```text
Preview:  prepare pool + reusable DDC slots -> open source -> PSD 1x
Run:      keep source/timer/PSD -> retarget prepared DDCs -> attach scanner
Clear:    cancel race/catch-up/locked futures -> discard decoder queue
          -> clear selections -> remain in PREVIEW
Stop:     cancel decoder -> close source -> STOPPED
EOF:      drain queued IQ -> graceful finalize -> COMPLETED
```

Run 会记录 `decoderRequestedSample` 和 `decoderReadySample`。默认路径中两者相同，
`sourceNextSample` 在 Run 调用前后也必须完全相同；这是“没有重开、回卷或
跳过文件”的回归不变式。

### 11.3 可复用 DDC 槽

`dsp.DigitalDownConverter.CenterFrequency` 是非 tunable 属性，选频后再构建并首次调用会
引入明显 JIT/滤波器初始化延时。新路径在预览开始前创建默认 5 个零中频
DDC 槽并锁定其滤波器；真实载波偏移由槽前的连续相位复本振完成。因此：

- Run 不修改 DDC 的非 tunable 属性；
- `ddcRetarget` 只重置相位、时间戳、缓冲和计数器；
- 原有 internal-mixer DDC 路径保留，不改变离线提取和其他入口的默认行为；
- external/internal 路径的基带输出实测归一化相关系数为 `1.000000000`。

2.5 MHz 下预备 5 个槽约需 10.3 s，这一次性工作在 PSD 开始前以
`PREPARING` 明示；三载波 Run 挂接基准为 0.132 s，隐藏 UI 实测为 0.167 s。
连续三载波 DDC 仍约为 7.6 ms/20 ms 输入块，没有因外部混频破坏 1x 余量。
真实 `synthesized_wideband_2.5MHz.rawiq` 五 worker 验收的 Run 挂接为
0.221 s，且仍正确得到 `DMR / DMR / P25` 三通道结果。

### 11.4 后备队列与清除语义

当用户显式关闭预热，或无法取得预备 DDC 槽时，仍保留异步构建后备路径。
Run 之后到挂接之前的 IQ 以 complex-single 存放在有界队列中，默认上限 15 s；
超限时明确取消解码并继续 PSD，不静默丢样。2.5 MHz 下该硬上限约对应
300 MB IQ 负载，它不是 60 MHz 生产队列方案。

Clear 不执行 tail flush，而是 best-effort 取消每个通道的 race、winner catch-up 和
locked decode future，释放 DDC，然后立即回到 `PREVIEW`。清除前后文件源样点
不变；后续计时器验收中 1.9 s 墙钟推进 1.9 s IQ，输入滞后为 0 ms。

### 11.5 MATLAB 热更新与强制清理

不应在 `radio_live_frontend` 窗口存活时替换函数文件。MATLAB 可能在文件更新后继续
投递旧 timer/异步 Channel 事件，从而使新回调访问旧 `state` 结构。当旧状态没有
`scannerBuildFuture` 等新字段时，旧版 Close 还会先设置 `closing=true`，导致后续关闭
永久早退。

当前实现为新字段提供惰性默认值，Close 不再因 `closing=true` 返回，并额外注册
`DeleteFcn` 作为计时器和解码任务的最终清理路径。如需清除修正前已经卡住的窗口，
在 MATLAB 命令行执行：

```matlab
figs = findall(groot, 'Type', 'figure', ...
    'Name', 'PMR 1x Live Multi-Carrier Frontend');
delete(figs);
timers = timerfindall('Name', 'PMRLeanLiveReplay');
if ~isempty(timers), stop(timers); delete(timers); end
clear radio_live_frontend
```

## 12. 五路近同步实时优化结果（2026-07-15）

在本记录的三路基线之后，新增的五路近同步样例暴露了 worker 饥饿、重复整窗传输、
五路独立 DDC 和 `fixedSpacing` 计时语义等问题。现已完成有界公平竞速、显示/活动检测
限频、调制家族门控、有界历史增量适配、矩阵融合 DDC，以及分类/catch-up 全局优先级。

严格五路运行的最终结果为：五个赢家全部与清单一致，逐路均产生 PDU，6.48 秒输入在
6.844 秒内完成，最大真实输入滞后 0.196 秒，解码队列峰值为 0，验收返回
`functionalPass=true` 和 `continuousDecodeRealtime=true`。完整阶段、命令、性能数据和
适用边界见[2.5 MHz 五路近同步信号合成与验收](2.5MHz五路近同步信号合成与验收.md)。

该结论只覆盖 2.5 MS/s 离线回放和五个已知频点，不扩展为 60 MHz 自动发现或 SDR
硬实时承诺。锁定解码内部仍是有界历史整窗兼容层，后续协议级持续状态化仍有优化空间。
