# 实时频谱选频与循环回放解码前端设计

> 实施状态（2026-07-15）：阶段 A～D 的离线文件原型已经完成并通过合成测试与真实
> BVSP 验收；阶段 E 只保留了统一 `IqChunk` 接口，等待真实 SDR 型号、驱动与数据格式
> 确定后实现。实际文件、接口和调试结论见
> `docs/实时频谱选频与循环回放解码前端实现记录.md`。
>
> 2026-07-15 更新：1x、多载波、自适应活动检测、异步 locked decode 与 2.5 MHz
> 三载波验收已经另行实现。本文中 `0.25x`、单载波和同步 locked decode 的描述仅代表
> 旧原型；当前结论见 `docs/2.5MHz一倍速多载波并行前端修正与验收记录.md`。

## 1. 目标

本设计在现有 `carrier_scope.m`、已知载频 DDC 和五制式并行识别基础上，增加一个可交互
的实验前端。用户能够观看持续更新的宽带功率谱和瀑布图，点击一个谱峰，随后点击
“运行识别”即可把该频点送入 DMR、P25、dPMR、NXDN、TETRA 并行竞争和赢家解码。

真实 SDR 型号、驱动和数据帧格式尚未确定，因此第一阶段以离线文件循环回放模拟连续
空口输入。文件源和未来 SDR 源必须使用相同的 `IqChunk` 接口，使频谱、DDC、协议竞争
和界面不依赖具体采集设备。

第一阶段目标：

1. BVSP、RAW 或双通道 WAV 文件可以连续或重复回放；
2. 前端以约 5～10 FPS 更新功率谱和瀑布图；
3. 点击谱峰后显示相对频偏，并在已知 RF 中心时显示绝对载频；
4. 点击“运行识别”后，从文件开头确定性重放并启动单载频 DDC；
5. 120 kS/s 基带直接进入流式 Epoch 和五制式 `RaceCoordinator`；
6. 界面持续显示分类状态、赢家、Epoch 和已解析 PDU；
7. 能用循环文件获得足够长的分类观测时间，并能验证重复 Epoch 行为。

第一阶段不承诺 61.44 MS/s 硬实时，不包含多载频同时解码，也不实现尚未确定的 SDR
驱动。它是功能正确性和交互流程原型。

## 2. 与现有模块的关系

`carrier_scope.m` 保留为静态离线频谱分析器，不继续堆叠实时状态、解码状态和数据源
生命周期。新增顶层入口建议命名为 `radio_frontend.m`。

```text
carrier_scope.m
  └─ 离线查看、平均谱、Max Hold、瀑布图、保存载频提示

radio_frontend.m
  └─ 连续文件/SDR输入、实时谱、交互选频、流式DDC、并行识别、PDU显示
```

可复用的频谱计算和点击精调逻辑应逐步从 `carrier_scope.m` 提取到 `+radio/+scope`，
避免两个界面各自维护一份FFT、频率轴和谱峰精调算法。

## 3. 总体数据流

```text
FileLoopSource / Future SdrSource
            |
            | IqChunk，10 ms，绝对样点号连续
            v
       Input Dispatcher
          /       \
         /         \
        v           v
Spectrum Engine   Selected-carrier Stream Scanner
  |                 |
  | 每100~200 ms     | 每个输入块
  v                 v
PSD / Max Hold     NCO DDC
/ Waterfall          |
  |                  v
  |             抗混叠滤波与抽取
  |                  |
  |                  v
  |              120 kS/s IqChunk
  |                  |
  |                  v
  |           Activity / RF Epoch
  |                  |
  |                  v
  |          Five-protocol RaceCoordinator
  |                  |
  |                  v
  |          Winner lock and continuous decode
  |                  |
  +------------------+
            |
            v
     UI status / Epoch / PDU table
```

频谱支路只消费抽样的显示快照，允许降低刷新率；DDC支路必须消费每一个输入块，并在块间
保持 NCO、CIC、FIR 和样点时间状态。不能为了减少绘图开销而跳过解码支路的数据。

## 4. 频率坐标与选频

宽带复数 IQ 已经相对于 SDR 本振进入数字基带，但目标载波不一定在 0 Hz。对 61.44 MHz
捕获，数字频率轴仍覆盖约 `-30.72 MHz ～ +30.72 MHz`。

界面始终维护：

```text
relative_offset_hz = clicked_frequency_on_baseband_axis
absolute_rf_hz = capture_center_frequency_hz + relative_offset_hz
```

DDC 只需要 `relative_offset_hz`。`capture_center_frequency_hz` 只负责绝对 RF 坐标、报告
和 PDU 元数据：BVSP 自动从头部读取；无头 RAW 可以不填，此时界面只显示相对频偏，
解码不受影响。

点击频谱后按以下步骤选频：

1. 鼠标位置作为粗频偏；
2. 在 6.25、12.5 或 25 kHz 信道能量窗口中进行局部中心精调；
3. 显示粗频点、精调频点、相对频偏和绝对 RF；
4. 用三角标记谱峰，不使用覆盖真实谱线的整条实线；
5. 解码运行期间锁定当前选择，换频必须停止并创建新的 DDC/Coordinator。

第一阶段只允许一个活动载频。界面可以保留多个书签，但“运行识别”只接受当前活动项。

## 6. 文件回放数据源

### 6.1 拉取和自然背压

文件阶段采用拉取模式。处理完当前块后才读取下一块，因此计算慢只会降低回放速度，
不会无限堆积内存，也不会丢文件数据。

```matlab
[source, chunk, done, event] = radio.replay.fileLoopSourceRead(source);
```

源状态至少包含：

- 文件路径、格式、采样率、头长度和捕获中心；
- 文件内样点位置和全局单调样点号；
- 循环次数、循环上限和回放模式；
- 下一序号、时间戳和 discontinuity 标志；
- 单次回放的真实样点数和累计逻辑样点数。

文件回到开头时，不能直接把 `sourceSampleStart` 重置为 0。源适配器应重新基准化为持续
递增的全局样点号，否则下游会把它当作时间倒退并重置环形缓冲。

### 6.2 回放模式

#### 单次回放 `once`

文件结束后冲洗 DDC 和 Epoch，停止识别。用于正常离线对比。

#### 连续测试 `continuous-test`

```text
EOF -> 文件开头
全局样点号继续
不主动声明 discontinuity
DDC/分类窗口继续积累
```

它用于把短文件重复为足够长的逻辑观测。例如默认五制式中 TETRA Probe 最大窗口为
6 秒，一秒文件至少循环 6 次，并额外留出初始化和边界裕量。

文件首尾不是物理连续波形，连接处会产生短暂瞬态，因此该模式必须在报告中标记
`synthetic_continuous_replay=true`，不能用来证明真实连续相位性能。合法帧在每次循环
内部保持不变，适合验证分类器能否积累到强证据。

#### 独立 Epoch `epoch-repeat`

```text
EOF -> 插入不少于 offHangSec 的零信号
     -> 关闭当前 RF Epoch
     -> 下一遍首块设置 discontinuity 或显式开始新 Epoch
```

它用于验证相同发射机重复上报、Epoch关闭、去重边界和重新分类。每次循环仍是独立观测，
不能用它把一秒文件扩展成六秒分类窗口。

## 7. 实时频谱引擎

输入块为 10 ms，但界面不应以 100 FPS 重绘。建议参数：

| 参数 | 建议值 |
|---|---:|
| 输入块 | 10 ms |
| PSD计算周期 | 50～100 ms |
| UI刷新周期 | 100～200 ms |
| UI帧率 | 5～10 FPS |
| FFT长度 | 65536，必要时131072 |
| 61.44 MS/s频率分辨率 | 937.5 Hz（NFFT=65536） |
| 瀑布图显示频点 | 4096～8192 |
| 瀑布图历史 | 100～300行 |

每次显示更新可以从最新块选若干不重叠 FFT 段，更新线性功率平均和 Max Hold，再把频谱
压缩到显示点数。界面只保留 PSD 行，不保留历史宽带 IQ。

绘图使用 `drawnow limitrate`，避免回调积压。预览计算和解码计算共用源块，禁止两个文件
句柄各自以不同速度前进，否则用户看到的谱与实际解码数据不一致。

## 8. 流式已知载频扫描器

现有 `radio.tuned.scanFile` 是“先完成整个DDC，再做离线多 Epoch 扫描”。交互前端需要
增加真正的逐块适配器：

```matlab
scanner = radio.tuned.streamScannerInit(inputFs, offsetHz, ...);
[scanner, output] = radio.tuned.streamScannerFeed(scanner, widebandChunk);
[scanner, final] = radio.tuned.streamScannerFinalize(scanner);
```

内部组合：

```text
radio.tuned.ddcFeed
  -> 120 kS/s IqChunk
  -> radio.stream.raceCoordinatorFeed
  -> events / selectedProtocol / closedEpochs / pdus
```

`streamScannerFeed` 只做单载频路由，不重新执行 PSD 候选发现。每个宽带块产生的零个或
多个基带块必须按顺序交给 Coordinator；输入间断时复用当前 DDC discontinuity 传播，
同时关闭旧 Epoch 并取消旧一代 Probe。

### 8.1 启动预热

当前 R2022b 的 `dsp.DigitalDownConverter` 首次调用会设计并锁定滤波器，五进程池首次
启动也有明显延迟。这些工作必须在频谱预览开始前的 `PREPARING` 阶段完成：

1. 创建或取得五 worker 进程池；
2. 创建默认五个可复用零中频 DDC 槽；
3. 向每个 DDC 输入全零固定块，完成滤波器设计与锁定；
4. `reset` DDC，清空预热输出和时间状态；
5. 打开文件源并开始 `PREVIEW` 1x 频谱。

点击“运行识别”后不再创建滤波器。选定频偏由 DDC 前的外部连续相位数字本振
完成，预备槽只重置时间和缓冲状态后挂接到当前样点。文件源、PSD 状态与回放
时钟都不得重置。

预热期间不能把虚构的零样点计入 Epoch。

### 8.2 停止条件

可配置：

- 手工停止；
- 确认赢家后继续解码指定秒数或循环数；
- 得到指定 PDU 数量；
- 达到最大逻辑回放时间；
- 所有 Probe 拒绝且达到最大窗口；
- 无活动达到超时。

默认测试建议：最大逻辑时间 10 秒；赢家锁定后继续 2 秒；未知信号在 TETRA 最大窗口
6 秒之外增加 0.5～1 秒裕量再结束。

## 10. MATLAB调度与性能策略

### 10.1 第一阶段文件原型

文件回放由单一调度循环拥有数据源、DDC和Coordinator，避免多个回调同时修改有状态 DSP
对象。每次只拉取一个块，处理完成后通过 `drawnow limitrate` 让出UI事件。

可以用短周期 `timer` 驱动，但必须设置 `BusyMode='drop'`，回调未结束时不能重入。文件
源是拉取式，即使定时器错过周期也不会丢数据。建议默认 `0.25x` 回放，用户可选择
`1x` 或 `unlimited`。

### 10.2 当前性能现实

现有实测中，61.44 MS/s 的 DDC 预热后处理 10 ms 块约需 14 ms；完整五制式离线流程
还包含进程池、分类和完整解码。加入FFT和GUI后，MATLAB CPU版本不能保证按10 ms墙钟
时间持续处理。

因此界面必须同时显示：

- 输入逻辑时间；
- 实际墙钟时间；
- 当前回放速度；
- 处理实时因子；
- 队列深度或文件拉取等待状态。

文件慢放属于正确行为，不算丢包。真实 SDR 不能反压硬件时才需要有界生产者/消费者
队列、溢出丢块和 discontinuity 上报。

### 10.3 真实SDR阶段

未来 SDR 源是推送式，建议结构：

```text
SDR callback / native reader
  -> bounded raw CI16 queue
  -> MATLAB consumer
  -> spectrum + selected DDC
```

队列只允许固定容量。溢出时丢弃最旧或最新块必须由策略明确，并记录丢失样点数；下一块
设置 `discontinuity=true`，让 DDC、环形缓冲和 Epoch 同步重置。不得通过无限堆积延迟
伪装成“无丢包实时”。

## 11. 内存预算

61.44 MS/s、10 ms：

```text
614400 complex samples per chunk
CI16原始块约2.46 MB
complex double块约9.83 MB
```

当前 DDC 因 R2022b 低幅度复数 `single` 异常暂用 `double`，文件拉取阶段只保留当前块和
至多一个余量块，不建立宽带历史队列。预览前的宽带 pre-trigger 默认不实现；点击运行
后文件回到开头，因此不会错过分类起点。

120 kS/s、10 ms仅1200个复数样点。现有8秒基带环形缓冲约96万个样点，使用复数
double时约15.4 MB。4096点、200行的single瀑布图约3.3 MB。总体内存主要由当前宽带
块和进程池worker占用，而不是基带缓存。

## 12. 异常和边界处理

- 未选频时，“运行识别”不可用；
- 选择超出输入 Nyquist 或滤波过渡带时立即拒绝；
- RAW无采样率时禁止预览；缺少RF中心只影响绝对频率显示；
- 文件首尾格式不完整时停止并报告，不静默丢弃半个IQ样点；
- 切换文件、采样率、频点或协议集合时销毁旧DDC和Coordinator；
- 输入断流关闭当前Epoch，取消正在运行的 Probe/Catch-up；
- 进程池不可用时允许明确标记的 `serial_fallback`；
- UI关闭时停止timer、取消任务、关闭文件句柄和释放DSP对象；
- PDU默认按Epoch去重，连续测试循环产生的重复帧应允许切换到“不去重”诊断模式。
