# 宽带流式前端阶段一实现记录

## 1. 本次实现结果

已在 `feature/streaming-protocol-race` 分支建立宽带流式前端。它不修改五种协议的
判决规则，而是在现有单频点流式竞速器外新增 `radio.wideband` 层：

```text
宽带 IqChunk
  -> 2x WOLA/PFB
  -> 活动粗带和细频点候选
  -> CandidateTracker
  -> 每候选连续细 DDC/低通
  -> 现有 RaceCoordinator
```

当前入口既能处理内存 IQ，也能从 RAW IQ/WAV 文件按 10 ms Chunk 回放；未来真实 SDR
只需持续调用同一个 `scannerFeed` 接口。

## 2. 新增模块

`+radio/+wideband`：

- `defaultConfig/validateConfig`：宽带、检测、跟踪、提取和窄带流配置；
- `channelizerInit/channelizerFeed`：连续 2x WOLA/PFB；
- `detectCandidates`：粗通道功率、局部 PSD、细载波和重复合并；
- `candidateTrackerInit/candidateTrackerFeed`：候选生命周期和频率平滑；
- `channelExtractorInit/channelExtractorFeed`：连续细 DDC、低通和宽带采样映射；
- `scannerInit/scannerFeed`：实时增量总入口；
- `scanIq`：内存 IQ 流式回放适配器；
- `scanFile`：不整文件加载的 RAW/WAV Chunk 回放入口；
- `emptyTrack/emptyChannel`：稳定的数据结构模板。

统一入口 `radio.scanFile` 新增：

```matlab
'ExecutionMode', 'wideband'
'WidebandConfig', radio.wideband.defaultConfig()
'CenterFrequencyHz', 430e6
```

`scanner.m` 也可以设置 `EXECUTION_MODE='wideband'`。宽带模式不调用整文件绘图路径，
避免把几十 GB 的采集文件一次性读入内存。

## 3. 连续性处理

WOLA/PFB 保存 `L-hop` 个重叠输入样点。每帧输出按照宽带窗口的绝对起点去除 FFT
调制相位，因此同一载波跨输入 Chunk 后仍是连续子带 IQ。细 DDC 保存 NCO 相位，
低通 FIR 保存 `zi/zf`。

PFB batch 同时记录：

```text
outputSampleStart / End
frameSourceSamples
widebandSampleRateHz
widebandCenterFrequencyHz
groupDelaySamples
continuityGeneration
```

协议 PDU 的 `extra.stream.source_sample` 通过提取器的采样率比例映射为
`extra.wideband.source_sample`，并附带 Track、RF 频率和粗通道信息。

## 4. Epoch 关系

每个候选频点创建独立 `RaceCoordinator`。窄带功率持续低于门限 300 ms 时，原有
`ChannelController` 关闭 Epoch。宽带 Track 默认保留 350 ms，确保关闭 Chunk 已送入
控制器后再释放提取器和协调器。

Track 不是 Session：

- 300 ms 静默关闭旧 Epoch；
- 350 ms 后宽带路由释放；
- 后续同频、同制式、同发射机重新出现时创建新的 Track 和 Epoch；
- 不进行跨 Epoch PDU 去重或业务归并。

## 5. 自动化测试

新增 `tests.runStreamingPhase10` 并纳入 `tests.runAll`，覆盖：

1. 61.44 MS/s 默认配置产生 60 kHz 通道间隔和 120 kS/s 子带；
2. 整段输入与任意两段 Chunk 输入的 PFB 输出一致；
3. 位于两个 PFB 通道正中间的载波在两路均保持约 0.8 幅度；
4. 同一个粗子带内相隔 12.5 kHz 的两个 4FSK 载波被分成两个候选；
5. TENTATIVE、ACTIVE、OFF_PENDING、短暂恢复和超时关闭；
6. 合成宽带 4FSK 候选接入现有 DMR RaceCoordinator、Epoch和持续解码；
7. PDU 带 RF 中心频率与宽带绝对采样位置；
8. 输入丢样/不连续时重置 PFB 输出编号并关闭旧候选；
9. `radio.scanFile(..., 'ExecutionMode','wideband')` 无信号文件入口。

## 6. 当前运行方式

### 6.1 `scanner.m`

```matlab
EXECUTION_MODE = 'wideband';
SAMPLE_RATE = 61.44e6;
FREQ_LIST = [];
BLIND_SEARCH = false;
PROTOCOLS = {};
WIDEBAND_CENTER_FREQUENCY_HZ = 430e6;
SHOW_FIGURE = false;
```

### 6.2 文件入口

```matlab
cfg = radio.wideband.defaultConfig();
[pdus, report] = radio.wideband.scanFile( ...
    '/path/capture_61440000.rawiq', ...
    'SampleRate', 61.44e6, ...
    'IqDType', 'int16', ...
    'CenterFrequencyHz', 430e6, ...
    'Config', cfg, ...
    'ProtocolNames', {}, ...
    'Mode', 'parallel');
```

### 6.3 未来 SDR 调用边界

```matlab
scanner = radio.wideband.scannerInit(61.44e6, 430e6);
while running
    chunk = sdrSourceRead(...);  % 必须符合 radio.stream.IqChunk 契约
    [scanner, output] = radio.wideband.scannerFeed(scanner, chunk);
    consume(output.newPdus);
end
```

SDR Chunk 必须给出连续的 `sourceSampleStart/sourceSampleEnd`、时间戳、中心频率、
`discontinuity` 和 `droppedSourceSamples`。

## 7. 已知限制

- 当前没有具体 SDR 驱动和生产级背压队列；
- MATLAB CPU PFB 尚未达到 60 MHz 实时速度；
- detector 门限只通过合成信号验证，尚未用真实宽带噪声、杂散和强弱信号标定；
- 离线快速回放尚未实现显式采集节流和候选准入上限；
- PFB 当前生成整块 `N x frames` 矩阵，后续可融合功率检测并只保留活动路；
- 同频同时发射无法由单天线信道化器普遍分离；
- 宽带层可以发现 25 kHz TETRA 载波，但 TMO 能否确认仍受协议解码器能力限制。

本阶段结论是“宽带到现有单频点并行链路的接口和正确性骨架已经建立”，不是
“60 MHz 纯 MATLAB 已达到在线实时性能”。
